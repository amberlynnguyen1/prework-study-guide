HTML

  • The head element contains information about the webpage.
  • The body element represents the visibility content shown to the user.
  • "id" is used as a way to identify elements and select them.
  • "ul" creates an unordered list of items.
  • "li" is used as bullet points under the "ul" element. Each one represents a new bullet point in our list.

CSS

  • A margin indicates how much spacce we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are 3 ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline CSS:
      1. Used when a developer wants to syle an element directly in the HTML file.
      • 2. Inline styles apply to a specific HTML tag, using a style attribute with a VSS rule to style a particular page element only.
  • Internal style sheet: contains CSS rules for the webpage in the head section of the HTML file.
  • External CSS Style Sheet:
      1. The most common way to style a webpage.
      2. Preferred in most cases because it allows developers to keep all their CSS riles in a separate file, which makes design changes easier.
  • declarations contain two important components: the CSS property we want to apply and the value of the property.
  • "Link" element creates the connection to the external file.
  • The "rel" attribute specifies the relationship between the current document and the linked document resouce.
  • The "href" attribute specifies the location (URL) of the external resouce.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout branch-name: allows you to switch to a branch already created.
  • git add -A: stages your changes.
  • git commit -m "add short description": allows you to associate a message with our commit.

JavaScript

  • Topics: contains a string that includees all of the topics that we are studying in this module
  • Control flow: the order in which a computer executes code in a script.
  • A value considered "falsy" if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false.
  • Strict Equiality Operator (===): checks to see if two values are equal, and return a Boolean result "true" if the values are equal and "false" if the values are not equal.
  • Array: a single variable that is used to hold a group of data.
